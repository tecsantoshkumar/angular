What is Manual Testing?

Manual testing is a software or application testing process in which high-level test cases are executed manually without using any automated tool. All test cases are executed by the tester manually to explore application issues and enhance the end user's perspective. This testing technique ensures whether the application or software is working, as mentioned in the requirement document or not. The reports of test cases are also generated manually.

Companies are also hiring testers who have knowledge of dynamic test design techniques to ensure the best end-user perspective on various applications. This software testing requires a wide knowledge of various tools and technologies. The difference between the expected output and the present output is defined as a defect. The developer fixed the defects and handed them to the tester for retesting.

Types of Testing

How do you perform Manual Testing ?

  • First, Tester read and understand all the required documents related to software, to select testing areas.

  • Tester observes requirement documents to cover all requirements specified by the clients.

  • Testers develop test case that cover all the documents of requirement.

  • All the test cases are executed manually by testers using testing techniques.

  • If bugs occurred then the testing team reports to the development team.

  • The Development team fixes bugs and handed the software to the testing team for a retest of software.
